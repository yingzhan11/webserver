Abaixo você encontrará alguns comandos de terminal que vão te ajudar a expandir os testes para nosso webserv.

1. Funcionamento

Execute o comando abaixo no terminal, e webserv deve ser capaz de funcionar com um arquivo padrao pre-definido.

./webserv

Execute o comando abaixo no terminal, para usar o curl.

curl -v http://localhost:8080

Execute o comando abaixo no terminal, para "multiple hostname".

curl -v -H "Host:site1" http://localhost:8080

Execute o comando abaixo no terminal, para testar metodo POST e "limit client body" (modificar no .config o client_max_body_size).

curl -X POST -H "Content-Type: text/plain" --data "OI" http://localhost:8080/curl.py

Execute o comando abaixo no terminal, para testar metodo DELETE.

curl -X DELETE http://localhost:8080/upload/curl

Execute o comando abaixo no terminal, para testar metodo GET.

curl -X GET http://localhost:8080/get.py

Execute o comando abaixo no terminal, para testar metodo desconhecido.

curl -X AFE http://localhost:8080/

Execute o comando abaixo no terminal, para testar metodo POST com chunked para imagem

curl --trace - --header "Transfer-Encoding: chunked" --header "Content-Type: image/jpeg" --data-binary @/home/ivbatist/Downloads/wallpaper.jpg http://localhost:8080/curl.py

Execute o comando abaixo no terminal, para testar metodo POST com chunked para texto

curl --trace - --header "Transfer-Encoding: chunked" --header "Content-Type: text/plain" --data "OICARALHO" http://localhost:8080/curl.py

Execute o comando abaixo no terminal, para testar metodo POST com chunked para video

curl --trace - --header "Transfer-Encoding: chunked" --header "Content-Type: video/mp4" --data-binary @/home/ivbatist/Downloads/videoTester.mp4 http://localhost:8080/curl.py

Execute o comando abaixo no terminal, para testar metodo POST para o CGI em loop infinito

curl -X POST -H "Content-Type: text/plain" --data "OI" http://localhost:8080/error.py

Execute o comando abaixo no terminal, para estressar o servidor, e ficarmos todos estressados juntos :D

siege -b -t30S -c10 http://localhost:8080/empty.html

Em outro terminal verificar o PID do servidor

ps aux | grep ./webserv

Monitorar uso da memoria

top -p PID
