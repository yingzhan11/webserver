Below you will find some terminal commands that will help you expand testing for our web server.

1. Functionality

Run the command below in the terminal, and the web server should be able to run with a predefined default configuration file.

./webserv

Run the command below in the terminal to use curl.

curl -v http://localhost:8080

Run the command below in the terminal to test multiple hostnames.

curl -v -H "Host:site1" http://localhost:8080

Run the command below in the terminal to test the POST method and the “client body limit” (change the client_max_body_size in the .config file).

curl -X POST -H "Content-Type: text/plain" --data "HI" http://localhost:8080/curl.py

Run the command below in the terminal to test the DELETE method.

curl -X DELETE http://localhost:8080/upload/curl

Run the command below in the terminal to test the GET method.

curl -X GET http://localhost:8080/get.py

Run the command below in the terminal to test an unknown method.

curl -X AFE http://localhost:8080/

Run the command below in the terminal to test the POST method with chunked transfer for an image.

curl --trace - --header "Transfer-Encoding: chunked" --header "Content-Type: image/jpeg" --data-binary @/home/ivbatist/Downloads/wallpaper.jpg http://localhost:8080/curl.py

Run the command below in the terminal to test the POST method with chunked transfer for text.

curl --trace - --header "Transfer-Encoding: chunked" --header "Content-Type: text/plain" --data "HELLOTHERE" http://localhost:8080/curl.py

Run the command below in the terminal to test the POST method with chunked transfer for a video.

curl --trace - --header "Transfer-Encoding: chunked" --header "Content-Type: video/mp4" --data-binary @/home/ivbatist/Downloads/videoTester.mp4 http://localhost:8080/curl.py

Run the command below in the terminal to test the POST method for a CGI script with an infinite loop.

curl -X POST -H "Content-Type: text/plain" --data "HI" http://localhost:8080/error.py

Run the command below in the terminal to stress test the server — so we can all be stressed together :D

siege -b -t30S -c10 http://localhost:8080/empty.html

In another terminal, check the server’s PID.

ps aux | grep ./webserv

Monitor memory usage.

top -p PID
